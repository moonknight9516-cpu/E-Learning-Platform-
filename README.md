## ğŸ“¡ API Documentation (Mock Layer)

The application uses a service-based architecture in `services/api.ts` that simulates a RESTful API with local persistence.

### Authentication
- `POST /api/auth/login`: Authenticates user by email.
- `POST /api/auth/signup`: Creates a new user profile.

### Courses
- `GET /api/courses`: Returns all courses with support for `search`, `category`, and `difficulty` filters.
- `GET /api/courses/:slug`: Fetches a single course by its URL slug.
- `POST /api/courses`: (Admin Only) Creates a new course.
- `PUT /api/courses/:id`: (Admin Only) Updates an existing course.
- `DELETE /api/courses/:id`: (Admin Only) Deletes a course.

### Enrollments
- `POST /api/enroll`: Enrolls the current user in a course.
- `GET /api/enrollments/:userId`: Fetches all enrollments for a specific user.
- `PUT /api/enrollments/:id/progress`: Updates completion status for a specific lesson and recalculates total progress.
